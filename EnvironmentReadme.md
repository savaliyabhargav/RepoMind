# RepoMind — Environment Setup & Progress Documentation

## What is RepoMind?
An AI-powered GitHub repository analyzer. Paste a GitHub URL and instantly understand the entire codebase — project overview, annotated file tree, deep dives, and a 5-mode AI chat (Bug Search, Enhancement Guide, Subsystem Explainer, Code Search, Impact Analysis).

## How to Start the Environment

```bash
# From D:\Projects\RepoMind
docker-compose up --build -d

# Check all containers are running
docker-compose ps

# Check backend logs
docker logs repomind-backend

# Stop everything
docker-compose down

# Stop and delete all data volumes (fresh start)
docker-compose down -v
```

## How Docker Networking Works Here

All containers are on `repomind-network`. They talk to each other using **service names as hostnames**:

- Backend → Postgres: `jdbc:postgresql://postgres:5432/repomind`
- Backend → Redis: `redis:6379`
- Backend → Kafka: `kafka:29092` (internal listener, not 9092)
- Backend → Qdrant: `qdrant:6333`
- Backend → MinIO: `http://minio:9000`

Port mappings like `5432:5432` only expose the service to your **host machine** (your Windows PC). Containers talk to each other directly through the Docker network without going through host ports.
